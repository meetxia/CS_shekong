# 测评页面布局优化记录

## 优化时间
2024年

## 优化需求
用户反馈桌面端测评页面左右两边没有留空位，内容布满整个屏幕，需要：
1. 桌面端左右各留17%空白，内容在中间65%区域居中显示
2. 导航栏、进度栏、内容区、底部操作栏都要保持一致的对齐
3. 禁止页面滚动，所有内容一屏显示完整
4. 移动端和iPad端保持原样，不做任何修改

## 优化方案

### 1. 桌面端布局调整
使用CSS媒体查询 `@media (min-width: 769px)` 区分桌面端，设置：
- 进度栏：padding 16px 17%
- 内容区：padding 24px 17%
- 操作栏：padding 16px 17%

### 2. 禁止滚动
- 设置 `.content-area` 的 `overflow-y: hidden`
- 移除桌面端的垂直滚动功能

### 3. 内容紧凑化
为适应一屏显示，对内容进行微调：
- 题目行高：1.6 → 1.4
- 题目底部间距：32px → 24px
- 选项间距：12px → 10px
- 选项内边距：20px → 16px 20px

### 4. 移动端保持不变
使用 `@media (max-width: 480px)` 保持原有移动端布局

## 修改文件

### src/views/AssessmentPage.vue
修改了以下样式规则：
1. `.progress-header` - 添加桌面端17%留白
2. `.content-area` - 添加桌面端17%留白和禁止滚动
3. `.question-text` - 行高和间距调整
4. `.options-list` - 间距调整
5. `.option-item` - 内边距调整
6. `.action-bar` - 添加桌面端17%留白

## 技术实现细节

### CSS媒体查询
```css
/* 桌面端 */
@media (min-width: 769px) {
  .progress-header { padding: 16px 17%; }
  .content-area { 
    padding: 24px 17%;
    overflow-y: hidden;
  }
  .action-bar { padding: 16px 17%; }
}

/* 移动端保持原样 */
@media (max-width: 480px) {
  .content-area { padding: 24px 16px; }
}
```

### 容器结构
```
.assessment-page (100vh, flex column, overflow hidden)
├── .progress-header (17% padding 桌面端)
├── .content-area (17% padding, overflow-y hidden 桌面端)
│   └── .question-container (max-width 600px, margin auto)
│       ├── .question-text
│       └── .options-list
│           └── .option-item × 5
└── .action-bar (17% padding 桌面端)
```
