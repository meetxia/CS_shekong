# 开发者调试面板使用说明

## 功能概述

在开发环境下（`npm run dev`），答题页面右上角会自动显示一个开发者调试面板，提供以下功能：

### 1. 清空所有答案
- **功能**：清除所有已选择的答案和 localStorage 中的缓存
- **用途**：快速重置测试状态，从头开始测试
- **按钮**：红色渐变按钮 "清空所有答案"

### 2. 随机填充答案
- **功能**：为所有 30 道题目随机选择答案
- **用途**：快速完成答题，测试提交流程和报告生成
- **按钮**：紫色渐变按钮 "随机填充答案"

### 3. 跳到第1题
- **功能**：快速跳转到第一题
- **用途**：从头开始测试
- **按钮**：蓝色渐变按钮 "跳到第1题"

### 4. 跳到第30题
- **功能**：快速跳转到最后一题
- **用途**：测试提交功能
- **按钮**：蓝色渐变按钮 "跳到第30题"

## 使用场景

### 场景1：测试选项选中动画
1. 点击 "清空所有答案"
2. 点击 "跳到第1题"
3. 逐个点击选项，观察动画效果

### 场景2：测试报告生成
1. 点击 "随机填充答案"
2. 点击 "跳到第30题"
3. 点击 "提交测评"
4. 观察报告生成流程

### 场景3：测试进度保存
1. 点击 "清空所有答案"
2. 随意选择几道题的答案
3. 刷新页面
4. 观察是否恢复到之前的进度

### 场景4：测试题目跳转
1. 点击 "随机填充答案"
2. 使用进度条拖动或点击跳转到不同题目
3. 观察答案是否正确显示

## 技术实现

### 自动检测开发环境
```javascript
const isDev = ref(import.meta.env.DEV)
```

- 只在开发环境（`npm run dev`）显示
- 生产环境（`npm run build`）自动隐藏
- 无需手动配置

### 清空答案实现
```javascript
const clearAllAnswers = () => {
  Object.keys(answers).forEach(key => {
    delete answers[key]
  })
  localStorage.removeItem('assessment_answers')
  showToast('已清空所有答案', 1500, 'success')
}
```

### 随机填充实现
```javascript
const fillRandomAnswers = () => {
  for (let i = 1; i <= 30; i++) {
    const question = questions.find(q => q.id === i)
    if (question) {
      const randomOption = question.options[Math.floor(Math.random() * question.options.length)]
      answers[i] = randomOption.id
    }
  }
  saveAnswers()
  showToast('已随机填充所有答案', 1500, 'success')
}
```

## 样式特点

- **半透明黑色背景** + 毛玻璃效果（backdrop-filter）
- **固定在右上角**，不影响正常答题
- **渐变色按钮**，视觉效果精致
- **悬停动画**，提升交互体验
- **高 z-index**，始终在最上层

## 注意事项

1. **仅开发环境可见**
   - 生产环境自动隐藏
   - 不会影响用户体验

2. **操作不可撤销**
   - 清空答案后无法恢复
   - 建议在测试前确认

3. **随机答案的随机性**
   - 每次点击都会生成新的随机答案
   - 可能覆盖已有的答案

4. **与正常功能兼容**
   - 不影响正常的答题流程
   - 不影响进度保存和恢复
   - 不影响报告生成

## 快捷键建议（未来扩展）

可以考虑添加键盘快捷键：
- `Ctrl + Shift + C`：清空所有答案
- `Ctrl + Shift + R`：随机填充答案
- `Ctrl + Shift + 1`：跳到第1题
- `Ctrl + Shift + 0`：跳到第30题
