# 测评页面布局优化说明

## 优化目标

根据用户反馈，对测评页面进行桌面端布局优化：
- 桌面端内容区域左右各留17%空白，中间65%区域显示内容
- 移除页面滚动，所有内容在一屏内显示
- 移动端和iPad端保持原样，不进行任何调整

## 具体修改

### 1. 进度栏优化
- 桌面端（≥769px）：左右padding设为17%，使进度栏与内容区域对齐

### 2. 内容区域优化
- 桌面端（≥769px）：
  - 左右padding设为17%，内容在65%区域内居中显示
  - 设置 `overflow-y: hidden` 禁止垂直滚动
  - 减少上下padding到24px，使内容更紧凑
- 移动端（≤480px）：保持原有的24px 16px padding

### 3. 题目和选项优化
- 题目文字行高从1.6调整为1.4，更紧凑
- 题目底部margin从32px调整为24px
- 选项列表gap从12px调整为10px
- 选项项padding从20px调整为16px 20px（上下更紧凑）

### 4. 底部操作栏优化
- 桌面端（≥769px）：左右padding设为17%，与内容区域对齐

## 响应式设计

```css
/* 移动端（≤480px）保持原样 */
@media (max-width: 480px) {
  .question-text { font-size: 20px; }
  .option-text { font-size: 16px; }
  .content-area { padding: 24px 16px; }
}

/* 桌面端（≥769px）应用17%留白 */
@media (min-width: 769px) {
  .progress-header { padding: 16px 17%; }
  .content-area { padding: 24px 17%; }
  .action-bar { padding: 16px 17%; }
}
```

## 视觉效果

### 桌面端
- 左右各17%留白，内容集中在中间65%区域
- 进度栏、内容区、操作栏完全对齐
- 无滚动条，所有内容一屏显示
- 更加专注和集中的阅读体验

### 移动端/iPad端
- 保持原有布局，无任何变化
- 继续支持滚动（小屏幕设备需要）

## 技术实现

- 使用CSS媒体查询 `@media (min-width: 769px)` 区分桌面端
- 使用CSS媒体查询 `@media (max-width: 480px)` 适配移动端
- 通过设置 `overflow-y: hidden` 禁止桌面端滚动
- 通过调整padding实现左右17%留白效果
