# 分数位置居中对齐修复

## 问题描述
报告页的分数标记位置不准确，63分显示位置偏右，没有准确对齐到63%的位置。

## 根本原因
`.your-position` 元素使用了 `position: absolute` 和 `left: 63%`，但这会让元素的**左边缘**对齐到63%的位置，而不是元素的**中心**。

### 示意图
```
错误的定位（left: 63%）：
|------------------------63%-----------100|
                        [63分]
                        ↑ 左边缘在63%

正确的定位（left: 63% + translateX(-50%)）：
|------------------------63%-----------100|
                      [63分]
                        ↑ 中心在63%
```

## 修复方案

### 修改前
```css
.your-position {
  position: absolute;
  color: var(--primary);
  font-weight: 700;
  font-size: 18px;
  /* 缺少居中对齐 */
}
```

### 修改后
```css
.your-position {
  position: absolute;
  color: var(--primary);
  font-weight: 700;
  font-size: 18px;
  transform: translateX(-50%); /* 添加此行，让元素中心对齐 */
}
```

## 技术说明

### `transform: translateX(-50%)` 的作用
- 将元素向左移动自身宽度的50%
- 使元素的中心点对齐到 `left` 指定的位置
- 这是居中对齐绝对定位元素的标准方法

### 完整定位逻辑
```vue
<span 
  class="scale-point your-position" 
  :style="{ left: `${report.totalScore}%` }"
>
  {{ report.totalScore }}
</span>
```

配合CSS：
```css
.your-position {
  position: absolute;
  left: [动态值]%;        /* 中心点的位置 */
  transform: translateX(-50%);  /* 向左偏移自身宽度的50% */
}
```

## 测试验证

修复后，不同分数的显示位置：

| 分数 | left值 | 实际显示位置 | 所在区间 |
|------|--------|-------------|---------|
| 30分 | 30% | 中心在30%位置 | 轻度（0-50%） |
| 50分 | 50% | 中心在50%位置 | 轻度/中度边界 |
| 63分 | 63% | 中心在63%位置 | 中度（50%-70%） |
| 70分 | 70% | 中心在70%位置 | 中度/重度边界 |
| 90分 | 90% | 中心在90%位置 | 重度/极重度边界 |

## 等级条宽度配置（正确）
- 轻度(0-50分)：50%宽度，占据0%-50%位置
- 中度(51-70分)：20%宽度，占据50%-70%位置
- 重度(71-90分)：20%宽度，占据70%-90%位置
- 极重度(91-100分)：10%宽度，占据90%-100%位置

## 修复状态
✅ 已修复 - 分数标记现在会精确居中对齐到对应的百分比位置
